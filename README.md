# StudyTracker

Десктопное приложение (Electron + TypeScript + React) для трекинга времени учёбы/работы,
которое **блокирует игры**, пока не достигнута дневная цель.

## Возможности

- Трекинг сессий (учёба / работа) с таймером и сохранением в локальную БД.
- Дашборд со статистикой за сегодня, вчера и последние 7 дней (графики).
- Гибкая дневная цель: отдельные значения для будней и выходных, настраиваемый час сброса.
- Блокировка игр: запрещённые игры автоматически закрываются, пока цель не достигнута,
  с показом уведомления.
- Добавление игр вручную (выбор `.exe`) или сканированием библиотеки Steam.
- Работа в системном трее и автозапуск при старте Windows.

## Как работает блокировка

Фоновый сервис каждые ~2 секунды опрашивает список запущенных процессов
(через `Get-CimInstance Win32_Process`). Если найден процесс из блок-листа, а дневная цель
ещё не выполнена, процесс немедленно завершается (`taskkill`). Для игр Steam совпадение
определяется по пути исполняемого файла внутри папки установки игры, для ручных записей — по
имени `.exe`.

> Это «дисциплинарная» блокировка для MVP. Чтобы её было сложнее обойти случайно, приложение
> продолжает работать в трее после закрытия окна, а при попытке выхода из трея во время активной
> блокировки показывается подтверждение.

## Защита от обхода (этап 5, на будущее)

MVP не защищает от целенаправленного обхода (можно выйти из трея, завершить процесс через
диспетчер задач и т.п.). Полноценная защита — отдельный этап, требующий прав администратора.
Исследованные варианты:

- **Windows-служба** (через `node-windows`/`nssm` или нативную службу): блокировщик работает как
  системная служба независимо от UI и автоматически перезапускается. Самый надёжный вариант, но
  требует установки с правами администратора.
- **Watchdog из двух процессов**: трекер и сторожевой процесс следят друг за другом и перезапускают
  упавшего. Не требует админ-прав, но пользователь всё ещё может завершить оба процесса.
- **Task Scheduler с наивысшими привилегиями**: перезапуск при входе в систему и по расписанию.
- **Friction в UI**: задержка/пароль на отключение блокировки, скрытие пункта «Выход» при активной
  блокировке.

Рекомендуемый путь для следующего этапа — вынести `BlockerService` в отдельную Windows-службу,
оставив текущее приложение как UI поверх неё.

## Технологии

- [electron-vite](https://electron-vite.org/) (Electron + Vite + TypeScript)
- React 19 + Tailwind CSS v4 + Recharts
- better-sqlite3 (локальное хранилище)

## Разработка

```bash
npm install      # установка зависимостей (+ пересборка нативного модуля под Electron)
npm run dev      # запуск в режиме разработки
npm run build    # сборка
npm run build:win # сборка установщика под Windows
```

Платформа: Windows.
